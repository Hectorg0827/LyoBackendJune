# Living Classroom - Client Integration Guide
## Real-Time Scene Streaming with WebSocket Architecture

> **Architecture Transformation**: From 60-120s HTTP polling to <1s real-time WebSocket streaming

---

## 🎯 Architecture Overview

```
iOS/Android Client ←→ WebSocket ←→ Scene Lifecycle Engine ←→ Multi-Agent System
    (Real-time UI)      (Bidirectional)   (Scene Director)      (Content Generation)
```

### Previous Architecture (Legacy)
```
Mobile Client → HTTP Request → Multi-Agent Pipeline → Full Course JSON → 60-120s latency
```

### New Living Classroom Architecture
```
Mobile Client → WebSocket → Trigger → Context → Director → Scene → <1s latency
```

## 🔄 Progressive Rendering

### Animation Timeline

1. **Scene Received**: Backend sends complete scene
2. **Components Queued**: Client queues components by priority
3. **Progressive Reveal**: Components appear with delays
4. **Animation In**: Each component animates in smoothly
5. **Interactive**: User can interact as components appear

### Implementation Example (React Native)
```tsx
const progressivelyRenderComponents = async (components: Component[]) => {
  const sortedComponents = [...components].sort((a, b) => a.priority - b.priority);

  for (const component of sortedComponents) {
    // Apply delay
    if (component.delay_ms) {
      await new Promise(resolve => setTimeout(resolve, component.delay_ms));
    }

    // Animate in
    setComponents(prev =>
      prev.map(comp =>
        comp.component_id === component.component_id
          ? { ...comp, isVisible: true }
          : comp
      )
    );
  }
};
```

## 🚀 Performance Optimizations

### Client-Side Optimizations

1. **Component Pooling**: Reuse component views
2. **Lazy Loading**: Load components only when visible
3. **Image Caching**: Cache media assets locally
4. **Connection Pooling**: Maintain persistent WebSocket

### Network Optimizations

1. **Message Compression**: gzip WebSocket messages
2. **Heartbeat**: 30-second ping/pong for connection health
3. **Reconnection Logic**: Automatic reconnection with exponential backoff
4. **Offline Support**: Queue actions when disconnected

### Fallback Strategy
1. **WebSocket Fails**: Fall back to HTTP polling
2. **Scene Fails**: Show generic "continue" button
3. **Auth Fails**: Redirect to login
4. **Network Fails**: Show offline state
